# Checklist antes de enviar

## 1. Links de contato (index.html, seção final)
- [ ] `mailto:seuemail@exemplo.com` → seu e-mail real
- [ ] `linkedin.com/in/seu-perfil` → seu LinkedIn
- [ ] `github.com/seu-usuario` → seu usuário do GitHub

## 2. Capturas de tela (pasta assets/)
- [ ] `assets/projeto-1-preview.png` — dashboard/briefing do Radar de Oportunidades
- [ ] `assets/projeto-2-preview.png` — fila de revisão do protótipo de saúde
- [ ] `assets/projeto-3-preview.png` — opcional, notebook da busca semântica
- [ ] `assets/projeto-4-preview.png` — print da plataforma Sprout

## 3. Conferir se o texto ainda bate com a realidade
- [ ] Projeto 1 (Radar de Oportunidades): confirme se o status "em produção" e a
      frase sobre integração com o Moskit ainda descrevem onde o projeto está
      no dia do envio — um avaliador técnico pode perguntar detalhe sobre isso
- [ ] Confirme que a nota de transparência sobre não incluir código-fonte
      proprietário do Projeto 1 (no README) ainda reflete o que você pode
      compartilhar publicamente

## 4. Nome de arquivo que o GitHub Pages precisa
- [ ] A landing precisa se chamar `index.html` na raiz do repositório —
      isso já está feito neste pacote, não renomeie de volta para portfolio.html

## 5. Publicar
- [ ] Criar repositório novo no GitHub (público)
- [ ] Subir todos os arquivos deste pacote mantendo a estrutura de pastas
      (index.html, README.md, prototipo-coaching-saude.html, assets/)
- [ ] Settings → Pages → Source: branch main, pasta / (root) → salvar
- [ ] Aguardar 1-2 minutos e testar o link gerado
      (formato: seu-usuario.github.io/nome-do-repositorio)
